8   # The routing module provides URL rewriting in native Ruby. It's a way to
 
9   # redirect incoming requests to controllers and actions. This replaces
 
10   # mod_rewrite rules. Best of all, Rails' Routing works with any web server.
 
11   # Routes are defined in <tt>config/routes.rb</tt>.
 
12   #
 
13   # Consider the following route, which you will find commented out at the
 
14   # bottom of your generated <tt>config/routes.rb</tt>:
 
15   #
 
16   #   match ':controller(/:action(/:id(.:format)))'
 
17   #
 
18   # This route states that it expects requests to consist of a
 
19   # <tt>:controller</tt> followed optionally by an <tt>:action</tt> that in 
20   # turn is followed optionally by an <tt>:id</tt>, which in turn is followed
 
21   # optionally by a <tt>:format</tt>
 
22   #
 
23   # Suppose you get an incoming request for <tt>/blog/edit/22</tt>, you'll end
 
24   # up with:
 
25   #
 
26   #   params = { :controller => 'blog',
 
27   #              :action     => 'edit',
 
28   #              :id         => '22'
 
29   #           }
 
30   #
 
31   # Think of creating routes as drawing a map for your requests. The map tells
 
32   # them where to go based on some predefined pattern:
 
33   #
 
34   #   AppName::Applications.routes.draw do |map|
 
35   #     Pattern 1 tells some request to go to one place
 
36   #     Pattern 2 tell them to go to another
 
37   #     ...
 
38   #   end
 
39   #
 
40   # The following symbols are special:
 
41   #
 
42   #   :controller maps to your controller name
 
43   #   :action     maps to an action with your controllers
 
44   #
 
45   # Other names simply map to a parameter as in the case of <tt>:id</tt>.

47   # == Named routes
 
48   #
 
49   # Routes can be named by passing an <tt>:as</tt> option,
 
50   # allowing for easy reference within your source as +name_of_route_url+
 
51   # for the full URL and +name_of_route_path+ for the URI path.
 
52   #
 
53   # Example:
 
54   #
 
55   #   # In routes.rb
 
56   #   match '/login' => 'accounts#login', :as => 'login'
 
57   #
 
58   #   # With render, redirect_to, tests, etc.
 
59   #   redirect_to login_url
 
60   #
 
61   # Arguments can be passed as well.
 
62   #
 
63   #   redirect_to show_item_path(:id => 25)
 
64   #
 
65   # Use <tt>root</tt> as a shorthand to name a route for the root path "".
 
66   #
 
67   #   # In routes.rb
 
68   #   root :to => 'blogs#index'

70   #   # would recognize http://www.example.com/ as

75   #   root_path  # => ''

137   # == HTTP Methods
 
138   #
 
139   # Using the <tt>:via</tt> option when specifying a route allows you to restrict it to a specific HTTP method.
 
140   # Possible values are <tt>:post</tt>, <tt>:get</tt>, <tt>:put</tt>, <tt>:delete</tt> and <tt>:any</tt>. 
141   # If your route needs to respond to more than one method you can use an array, e.g. <tt>[ :get, :post ]</tt>. 
142   # The default value is <tt>:any</tt> which means that the route will respond to any of the HTTP methods.
 
143   #
 
144   # Examples:
 
145   #
 
146   #   match 'post/:id' => 'posts#show', :via => :get
 
147   #   match 'post/:id' => "posts#create_comment', :via => :post

150   # URL will route to the <tt>show</tt> action. 
151   #
 
152   # === HTTP helper methods
 
153   #
 
154   # An alternative method of specifying which HTTP method a route should respond to is to use the helper
