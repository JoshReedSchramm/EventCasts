14     # Rails methods (that is, making a background request to a Rails action) using Ajax.
 
15     # This means that you can call actions in your controllers without
 
16     # reloading the page, but still update certain parts of it using

51     # == Designing your Rails actions for Ajax
 
52     # When building your action handlers (that is, the Rails actions that receive your background requests), it's
 
53     # important to remember a few things.  First, whatever your action would normally return to the browser, it will
 
54     # return to the Ajax call.  As such, you typically don't want to render with a layout.  This call will cause
 
55     # the layout to be transmitted back to your page, and, if you have a full HTML/CSS, will likely mess a lot of things up.
 
56     # You can turn the layout off on particular actions by doing the following:
 
57     #
 
58     #  class SiteController < ActionController::Base
 
59     #    layout "standard", :except => [:ajax_method, :more_ajax, :another_ajax]
 
60     #  end
 
61     #
 
62     # Optionally, you could do this in the method you wish to lack a layout:
 
63     #
 
64     #  render :layout => false
 
65     #
 
66     # You can tell the type of request from within your action using the <tt>request.xhr?</tt> (XmlHttpRequest, the
 
67     # method that Ajax uses to make background requests) method.
 
68     #  def name
 
69     #    # Is this an XmlHttpRequest request?
 
70     #    if (request.xhr?)
 
71     #      render :text => @name.to_s
 
72     #    else
 
73     #      # No?  Then render an action.

79     # to this solution was posted to Ryan Heneise's blog at ArtOfMission["http://www.artofmission.com/"].
 
80     #
 
81     #  layout proc{ |c| c.request.xhr? ? false : "application" }
 
82     #
 
83     # Dropping this in your ApplicationController turns the layout off for every request that is an "xhr" request.
 
84     #
 
85     # If you are just returning a little data or don't want to build a template for your output, you may opt to simply
 
86     # render text output, like this:

88     #  render :text => 'Return this from my method!'
 
89     #
 
90     # Since whatever the method returns is injected into the DOM, this will simply inject some text (or HTML, if you

813         # Options
 
814         #   * variable - name of the variable to set the result of the enumeration to
 
815         #   * method_args - array of the javascript enumeration method args that occur before the function
 
816         #   * yield_args - array of the javascript yield args
 
817         #   * return - true if the enumeration should return the last statement
 
818         def enumerate(enumerable, options = {}, &block)
 
819           options[:method_args] ||= []
 
820           options[:yield_args]  ||= []
 
821           yield_args  = options[:yield_args] * ', '
 
822           method_args = arguments_for_call options[:method_args] # foo, bar, function
 
823           method_args << ', ' unless method_args.blank?
 
824           add_variable_assignment!(options[:variable]) if options[:variable]
 
825           append_enumerable_function!("#{enumerable.to_s.camelize(:lower)}(#{method_args}function(#{yield_args}) {")
 
826           # only yield as many params as were passed in the block
 
827           yield(*options[:yield_args].collect { |p| JavaScriptVariableProxy.new(@generator, p) }[0..block.arity-1])
 
828           add_return_statement! if options[:return]
