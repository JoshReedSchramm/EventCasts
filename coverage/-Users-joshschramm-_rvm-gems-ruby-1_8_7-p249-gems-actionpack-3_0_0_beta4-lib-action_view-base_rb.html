15   #
 
16   # You trigger ERb by using embeddings such as <% %>, <% -%>, and <%= %>. The <%= %> tag set is used when you want output. Consider the
 
17   # following loop for names:
 
18   #
 
19   #   <b>Names of all the people</b>
 
20   #   <% for person in @people %>
 
21   #     Name: <%= person.name %><br/>
 
22   #   <% end %>
 
23   #
 
24   # The loop is setup in regular embedding tags <% %> and the name is written using the output embedding tag <%= %>. Note that this
 
25   # is not just a usage suggestion. Regular output functions like print or puts won't work with ERb templates. So this would be wrong:

27   #   <%# WRONG %>
 
28   #   Hi, Mr. <% puts "Frodo" %>
 
29   #
 
30   # If you absolutely must write from within a function use +concat+.

34   # == Using sub templates
 
35   #
 
36   # Using sub templates allows you to sidestep tedious replication and extract common display structures in shared templates. The
 
37   # classic example is the use of a header and footer (even though the Action Pack-way would be to use Layouts):
 
38   #
 
39   #   <%= render "shared/header" %>
 
40   #   Something really specific and terrific
 
41   #   <%= render "shared/footer" %>
 
42   #
 
43   # As you see, we use the output embeddings for the render methods. The render call itself will just return a string holding the
 
44   # result of the rendering. The output embedding writes it to the current template.
 
45   #
 
46   # But you don't have to restrict yourself to static includes. Templates can share variables amongst themselves by using instance
 
47   # variables defined using the regular embedding tags. Like this:
 
48   #
 
49   #   <% @page_title = "A Wonderful Hello" %>
 
50   #   <%= render "shared/header" %>
 
51   #
 
52   # Now the header can pick up on the <tt>@page_title</tt> variable and use it for outputting a title tag:
 
53   #
 
54   #   <title><%= @page_title %></title>

58   # You can pass local variables to sub templates by using a hash with the variable names as keys and the objects as values:
