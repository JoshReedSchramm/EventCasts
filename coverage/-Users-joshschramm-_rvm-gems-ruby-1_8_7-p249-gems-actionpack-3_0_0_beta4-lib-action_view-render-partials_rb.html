9   # In a template for Advertiser#account:
 
10   #
 
11   #  <%= render :partial => "account" %>
 
12   #
 
13   # This would render "advertiser/_account.erb" and pass the instance variable @account in as a local variable

15   #
 
16   # In another template for Advertiser#buy, we could have:
 
17   #
 
18   #   <%= render :partial => "account", :locals => { :account => @buyer } %>

21   #     <%= render :partial => "ad", :locals => { :ad => ad } %>
 
22   #   <% end %>
 
23   #
 
24   # This would first render "advertiser/_account.erb" with @buyer passed in as the local variable +account+, then

27   # == Rendering a collection of partials
 
28   #
 
29   # The example of partial use describes a familiar pattern where a template needs to iterate over an array and
 
30   # render a sub template for each of the elements. This pattern has been implemented as a single method that
 
31   # accepts an array and renders a partial by the same name as the elements contained within. So the three-lined
 
32   # example in "Using partials" can be rewritten with a single line:
 
33   #
 
34   #   <%= render :partial => "ad", :collection => @advertisements %>
 
35   #
 
36   # This will render "advertiser/_ad.erb" and pass the local variable +ad+ to the template for display. An
 
37   # iteration counter will automatically be made available to the template with a name of the form
 
38   # +partial_name_counter+. In the case of the example above, the template would be fed +ad_counter+.
 
39   #
 
40   # NOTE: Due to backwards compatibility concerns, the collection can't be one of hashes. Normally you'd also
 
41   # just keep domain objects, like Active Records, in there.
 
42   #
 
43   # == Rendering shared partials
 
44   #
 
45   # Two controllers can share a set of partials and render them like this:
 
46   #
 
47   #   <%= render :partial => "advertisement/ad", :locals => { :ad => @advertisement } %>
 
48   #
 
49   # This will render the partial "advertisement/_ad.erb" regardless of which controller this is being called from.
 
50   #
 
51   # == Rendering objects with the RecordIdentifier

83   # == Rendering partials with layouts
 
84   #
 
85   # Partials can have their own layouts applied to them. These layouts are different than the ones that are
 
86   # specified globally for the entire action, but they work in a similar fashion. Imagine a list with two types
 
87   # of users:
 
88   #
 
89   #   <%# app/views/users/index.html.erb &>

104   #

107   #     Deadline: <%= user.deadline %>

109   #   </div>
 
110   #
