7     # Page caching is an approach to caching where the entire action output of is stored as a HTML file that the web server
 
8     # can serve without going through Action Pack. This is the fastest way to cache your content as opposed to going dynamically
 
9     # through the process of generating the content. Unfortunately, this incredible speed-up is only available to stateless pages
 
10     # where all visitors are treated the same. Content management systems -- including weblogs and wikis -- have many pages that are
 
11     # a great fit for this approach, but account-based systems where people log in and manipulate their own data are often less
 
12     # likely candidates.
 
13     #
 
14     # Specifying which actions to cache is done through the <tt>caches_page</tt> class method:
 
15     #
 
16     #   class WeblogController < ActionController::Base
 
17     #     caches_page :show, :new
 
18     #   end
 
19     #
 
20     # This will generate cache files such as <tt>weblog/show/5.html</tt> and <tt>weblog/new.html</tt>,
 
21     # which match the URLs used to trigger the dynamic generation. This is how the web server is able
 
22     # pick up a cache file when it exists and otherwise let the request pass on to Action Pack to generate it.
 
23     #
